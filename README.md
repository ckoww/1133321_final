# 飯店房務與客房紀錄管理系統 (Hotel Room Management System)

透過直觀的視覺化介面，讓使用者能快速掌握各客房的即時狀態，並完成房務交接與退房日期的追蹤，有效提升飯店客房管理的營運效率。

### ✨ 核心功能與特色
* **視覺化房況總覽 (Dashboard)**：系統啟動時會動態生成客房矩陣，並以直覺的顏色（綠色空房、紅色入住、黃色待清潔、灰色維修）標示房況，一目了然。
* **智慧防呆與 UI 連動機制**：當房間切換為「空房」或「維修中」時，系統會自動鎖定並重置「退房日期」與「客房服務指示」，防止人員誤填不合理的交接資訊。
* **客房服務狀態追蹤**：支援「請勿打擾 (DND)」與「請即打掃 (MUR)」標籤，並直接顯示於房況總覽按鈕上，無須點擊即可獲取房務指示。
* **穩定的資料持久化 (SQLite)**：採用 `System.Data.SQLite` 進行本機資料庫存取，具備完善的資料庫初始化與防錯機制。

---

## 🚀 開發環境與技術
* **開發語言**：C# (.NET Framework)
* **UI 框架**：Windows Forms (WinForms)
* **資料庫**：SQLite (`System.Data.SQLite.Core`)
* **架構特色**：動態控制項生成 (Dynamic Control Creation)、自適應網格排版 (TableLayoutPanel)

## 💻 執行說明 


功能介紹
視覺化房況總覽 (動態按鈕矩陣)

系統啟動時會從資料庫動態讀取客房資料，並自動於左側面板生成對應的客房按鈕（預設為 101~106 號房）。

顏色直覺辨識：按鈕會根據當前房況自動變換背景顏色：

🟢 空房：淺綠色 (LightGreen)

🔴 入住中：淺紅色 (LightCoral)

🟡 待清潔：卡其色/黃色 (Khaki)

⚪ 維修中：淺灰色 (LightGray)

房務指示圖示：若客房有特殊服務需求，按鈕上會直接顯示對應圖示（如「🚫 DND」代表請勿打擾、「🧹 需打掃」代表請即打掃），讓房務人員無須點開詳細資料即可一目了然。

客房詳細資料編輯與連動防呆

點擊左側任一客房按鈕後，右側編輯區會自動解鎖並載入該房間的詳細資訊，包含：房號、當前房況、退房日期、服務指示與客房備註。

UI 狀態連動 (防呆機制)：當使用者將房況切換為「空房」、「待清潔」或「維修中」時，系統會自動鎖定（反灰）「預計退房日期」與「客房服務指示」欄位，並將服務指示強制重置為「無指示」，避免人員誤填無效的交接資訊。

日期保護機制：退房日期選擇器（DateTimePicker）限制最低只能選擇「今天」，防止人員誤選過去的日期；同時，若資料庫內留存有舊的過期日期，系統讀取時會自動將其安全轉換為當前日期，避免程式崩潰。

單鍵儲存與安全取消

儲存修改：使用者在右側編輯完畢後，按下儲存按鈕即可將新狀態與備註同步寫入 SQLite 資料庫，左側的房況總覽面板也會立即更新顏色與圖示。

安全取消：若編輯到一半想放棄修改，點擊取消按鈕即可一鍵清空所有輸入框，並將右側編輯區重新上鎖，避免誤觸修改到其他房間的資料。

自動化資料庫建置

具備 SQLite 資料庫自動初始化能力。程式首次執行時，若偵測到環境中沒有 .db 檔案或資料表為空，系統會自動建立標準資料表格式，並匯入初始的空房測試資料，達到隨插即用的效果。
